## Codebase Patterns

- **Pre-existing test failures**: 16 tests in `tests/task_queue/` and `tests/test_config.py` fail on this branch before any changes. These are not caused by feature work.
- **Click as main dep**: After task-005, click is added as a main dependency (not just dev). `uv sync --extra dev` still installs everything needed.
- **Click not in project deps yet**: Click is only added as a dev dependency so far (task-003 will add it as a main dependency). Test files importing `click.testing.CliRunner` need click in dev extras.
- **pytest runs with `uv run python -m pytest` + `--extra dev`**: `uv sync --extra dev` installs pytest, click, etc.
- **_import_cli() pattern**: Integration tests use a helper that returns None if `sokrates.cli.main` doesn't exist yet; tests skip gracefully using `pytest.skip()`. This allows the test file to be collected even before the implementation exists.
- **conftest.py module aliasing**: `tests/conftest.py` maps `src.sokrates` → `sokrates` in sys.modules and adds `src/` to sys.path.
- **Lazy loading is critical**: `sokrates_daemon` has module-level `CONFIG = Helper.load_config()` that reads from disk. If `main.py` imports modules at load time and `SOKRATES_HOME_PATH` points to a deleted temp dir (left by `test_config.py`), all unit tests fail. Always use lazy loading — import modules only when a command is actually invoked, not at `main.py` module level.
- **Passthrough commands for argparse**: Use `_make_lazy_command()` pattern: Click command with `add_help_option=False`, `allow_extra_args=True`, `ignore_unknown_options=True`. Intercept `--help` to show Click epilog first, then delegate to argparse `--help`. For normal invocations, set `sys.argv = [f'sokrates {name}'] + args` before calling `main_fn()`.
- **cmd_obj.help vs short_help**: Tests check `cmd_obj.help` (not `short_help`). Set the `help` parameter in `@click.command(help=short_help)` so `cmd_obj.help` is populated. The `short_help` attribute is derived from `help` by Click automatically.
- **Test isolation with SOKRATES_HOME_PATH**: `test_config.py` sets the env var but doesn't restore it (pre-existing bug). Lazy loading prevents this from breaking subsequent test collection.
